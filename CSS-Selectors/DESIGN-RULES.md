# قوانین طراحی وب: تصاویر و تصویرسازی‌ها (راهنمای جامع و فنی)

## ۱. روان‌شناسی بصری و اصول دیزاین (Visual Psychology)
* **هدف اصلی:** تصاویر فوراً احساسات کاربر را درگیر می‌کنند (در حالی که متن منطق را فعال می‌کند) و داستان کیفیت برند را روایت می‌سازند.
* **قانون جهت نگاه (Gaze Direction):** اگر در تصویری از چهره انسان استفاده شده است، مطمئن شوید که جهت چشم‌های او به سمت عنوان اصلی یا دکمه اکشن (CTA) سایت باشد. کاربران به طور طبیعی مسیر نگاه سوژه را دنبال می‌کنند. اگر نگاه سوژه به بیرون صفحه باشد، حواس کاربر پرت می‌شود.
* **سادگی بصری و کراپ کردن (Cropping):** از پس‌زمینه‌های شلوغ دوری کنید. بخش‌های اضافه را با کراپ کردن هوشمندانه حذف کنید تا سوژه اصلی در کمتر از ۱ ثانیه قابل تشخیص باشد.
* **ممنوعیت عکس‌های فیکِ استاک (Cheesy Stock Photos):** هرگز از عکس‌های اداری تصنعی با لبخندهای فیک استفاده نکنید. این کار حس ناامنی داده و اعتماد کاربر را نابود می‌کند. همیشه از عکس‌های طبیعی و واقعی استفاده کنید.

## ۲. انتخاب بین عکس واقعی و تصویرسازی (Images vs. Illustrations)
* **عکس‌های واقعی (Real Images):** برای محصولات فیزیکی (مثل صندلی لیسبون، لباس، غذا) و چهره انسان‌ها (در بخش نظرات مشتریان یا Testimonials) برای جلب اعتماد استفاده می‌شود.
* **تصویرسازی‌های گرافیکی (Illustrations):** برای محصولات دیجیتال، اپلیکیشن‌ها، نرم‌افزارها (SaaS) یا مفاهیم انتزاعی تکنولوژی که وجود فیزیکی ندارند استفاده می‌شود و حس مدرن و صمیمی به سایت می‌دهد.

## ۳. یکپارچه‌سازی فنی و استانداردهای وب (Technical Details)
1. **هارمونی رنگ (Color Harmony):** رنگ‌های غالب درون تصاویر شما باید با پالت رنگی CSS وب‌سایت کاملاً هماهنگ و ست باشند.
2. **کنتراست متن روی عکس (Text Overlay):** متن‌هایی که روی عکس قرار می‌گیرند باید کاملاً خوانا باشند. همیشه در صورت نیاز از یک لایه تیره نیمه‌شفاف با کد CSS مثل `rgba(0,0,0,0.5)` استفاده کنید تا متن سفید به‌خوبی دیده شود.
3. **قانون نمایشگرهای رتینا (Retina Displays - 2x):** برای اینکه تصاویر در نمایشگرهای باکیفیت (مثل مک‌بوک و آیفون) تار به نظر نرسند، ابعاد واقعی فایل عکسی که دانلود می‌کنید باید دقیقاً **۲ برابر (2x)** ابعادی باشد که در CSS برای آن باکس در نظر گرفته‌اید (مثال: برای باکس ۴۰۰ پیکسلی، عکس با عرض ۸۰۰ پیکسل مناسب است).
4. **بهینه‌سازی فرمت‌ها برای سرعت سایت:**
   * `.svg`: گزینه‌ای ایده‌آل برای آیکون‌ها و گرافیک‌ها. برداری است و در مانیتورهای باکیفیت هرگز کیفیت خود را از دست نمی‌دهد.
   * `.webp`: استاندارد مدرن وب از طرف گوگل. همیشه فایل‌های سنگین `.jpg` و `.png` را قبل از آپلود در پروژه به `.webp` تبدیل کنید تا سرعت لود صفحه حفظ شود.
   * `.png`: فقط زمانی استفاده شود که پس‌زمینه کاملاً شفاف (Transparent) مورد نیاز باشد (مثل لوگوها).

## ۴. فرمول‌های طلایی CSS برای مدیریت تصاویر

### الف) قانون طلایی واکنش‌گرایی (Responsive Images)
برای جلوگیری از بیرون زدن تصاویر از باکس خود و حفظ تناسب ابعاد، همیشه این قوانین پایه را روی تصاویر اعمال کنید:
```css
img {
  width: 100%;       /* عکس از باکس والد خود بیرون نمی‌زند */
  height: auto;      /* نسبت ابعاد عکس حفظ و دفرمه نمی‌شود */
  display: block;    /* حذف فاصله خالی و نامرئیِ زیر عکس */
}
img {
  width: 100%;       /* عکس از باکسش بیرون نمی‌زند */
  height: auto;      /* نسبت ابعاد عکس حفظ می‌شود */
  display: block;    /* حذف فاصله خالی ریز زیر عکس */
}
/* اگر مجبور به دادن عرض و ارتفاع ثابت شدی: */
.product-img {
  width: 400px;  height: 300px;
  object-fit: cover; /* برش هوشمند بدون کشیده یا له شدن عکس */
}
